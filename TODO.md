- **模块**：财务 / 费用管理
- **背景**：用户需一次性为所有未建账单的费用批量创建账单，现有逐一点击方式效率极低
- **任务**：
  - 确认"批量审批/审核费用"功能是否已存在或被误删
  - 新增"批量生成账单"接口，对指定范围内所有未建账单的费用一次性创建账单

### T6. 新功能：空运出口「主单领用登记」三表结构
- **模块**：空运出口
- **背景**：管理从航司领取的 MAWB，区分"登记"（直接从航司领）和"过单"（从代理借）
- **数据模型**：主单台账(MAWB_Stock) + 领入记录(Stock_In) + 领出记录(Stock_Out) 三表
- **MAWB规则**：3位航司前缀 + "-" + 8位数字（末位为校验位，前7位 mod 7 得出）；数据库存储标准化无空格号码
- **任务**：
  - 创建三个实体类及对应 DbSet
  - 实现 MawbStockManager（或在现有 MawbManager 中扩展）相关业务逻辑
  - 实现领入批量操作、领出单张操作接口
  - 在工作号选择主单时，从台账"未使用"状态中选取

### T7. 新功能：空运进口「进口仓单」主/子表实体及CRUD接口
- **模块**：空运进口
- **背景**：用于海关报关，本期只做手动CRUD
- **数据要点**：
  - 主单号为 **11位纯数字**（与航空公司标准格式不同），原样存储
  - 状态时间字段（运抵/理货/入库）精确到分钟，为空表示未发生
  - 子表中分单号为空的行表示主单信息行
- **校验逻辑**：所有行必须有主单号；同一主单下所有分单的件/重/体之和须等于主单行数据
- **任务**：
  - 创建进口仓单主表实体 + 子表实体 + 对应 DbSet
  - 实现标准 CRUD 控制器接口
  - 实现手动录入时的校验逻辑

### T8. 新功能：海运出口「海运提单」主/分单实体及CRUD接口
- **模块**：海运出口
- **背景**：周期长，单据通常独立于工作号早期创建（占单）
- **数据要点**：
  - 所有字段默认字符串、可为空，后端除明确指定外不作校验（支持先占单后完善）
  - 港口使用港口全名，非三字代码
  - 交通工具由"船名"和"航次"两个字段组成
  - 分提单字段结构与主提单完全一致，可复用实体
  - 分提单与主提单通过**工作号**建立逻辑联系，非数据库外键
- **关联关系**：一个工作号关联一个主提单；一个工作号关联多个分提单
- **任务**：
  - 创建主提单实体（EsMbl）和分提单实体（EsHbl）及对应 DbSet
  - 实现标准 CRUD 控制器接口
  - 明确与 PlJob（工作号）的关联字段
