权限设置页：窗口左右分栏，左侧提供目录树（例如「基础数据」→「客户资料管理」），点击可定位跳转，形态类似资源管理器。
菜单层级：支持到 三级菜单；二级节点既可进入页面，也可展开第三级。展开策略：点击「业务管理」时 2/3 级同时展开；样式用缩进（必要时以空格实现）。
前端控件：已有面板更换为新控件集，尚未全部替换完成。
2）权限调整（既有流程）
「1.3 权限」应放置在 费用申请单获取接口（例如 getAll…）而非工作流（WF）接口；WF 接口用于审批人视角，无需额外限制。
含义：无该权限仅可见「自己」的申请；拥有该权限可查看「所有人」的申请。
3）外部流程总览
现有「对内」流程：委托→费用→审批→结算→财务统计。
新增「对外」流程：主单、分单、舱单、报关；本期聚焦主单领用登记。
4）主单领用登记（核心）
4.1 业务背景
航司向 一级代理发放主单（类似发票管理），代理需按月报备领用/使用/转移/作废情况。
领用来源两类：登记（直接向航司领）与 过单（向其他代理借领）。
4.2 数据模型（表设计）
建议将 领入 与 领出 拆分为两张表（或采用「主单台账 + 领入 + 领出」三表），以更清晰地覆盖互斥字段与提交流程；用「主单号」建立关联。
状态：未使用 / 已使用 / 已作废；其中「已使用/未使用」可通过是否被业务占用来推导，仅「作废」建议在台账中显式记录（是否冗余回写「已使用」以便检索可后续评估）。
4.3 关键字段（领入）
主单号（唯一，见第 5 节校验规则）
来源：航司登记 / 过单代理
航空公司（从客户资料选择）
过单代理（供应商/代理；从资料表选择）
登记日期、创建者、创建时间等
4.4 关键字段（领出）
主单号、领单公司（客户，通常为二级代理）
领用人（自由文本，例：张三/李四）
领出日期、预计返单日期
4.5 批量与交互
领入：支持批量生成（前缀 + 起始 8 位 + 数量），可通过多次生成实现断号跳过。
领出：不批量；对单张主单执行领出登记。
纠错策略：支持修改，或采用「删除后重建」。
4.6 与业务单（委托）的关联
在业务界面通过选择「未使用主单」完成绑定；使用后可由主单号回查委托信息，在列表显示件数/重量/体积/计费重量等（读取显示，不写回）。
4.7 权限
建议权限颗粒度：8 项（领入 C/R/U/D ×4 + 领出 C/R/U/D ×4），通过角色组合授予。
5）主单号规则（校验与生成）
结构：3 位航司前缀 + “-” + 8 位数字（末位为 0–6 的校验位）。
校验：将前 7 位视为整数，减去第 8 位，结果对 7 取模为 0 则合法。
生成下一个号：前 7 位 +1 并重算校验位（或整体 +1 后对末位取 7 进制轮转）；不足 8 位时左侧补零。
格式差异：部分航司使用 4-4 分隔（中间空格）。输入校验需兼容空格；存储采用标准化主单号（去空格） + display 字段保存原始显示格式；界面显示优先使用 display。
前缀校验：暂不强制校验 3 位前缀合法性（后续可引入航司前缀字典）。
6）前端界面与交互细节
领入弹窗：航司为必填；勾选「过单」则必须选择「过单代理」。
起始主单号输入分为「前缀（3 位）」「—（默认）」「后 8 位」；校验起始号合法性后按数量批量生成。
列表页：展示台账字段及回查的业务指标；支持 代理多选（OR） 查询。
显示策略：组件层展示 display 主单号；内部逻辑一律使用标准主单号。
只读/编辑：按权限控制；若仅浏览，可不提供详情页。
7）菜单/信息架构调整
建议在「业务管理」下增设「单据操作」分组，包含：主单领用登记、主单制作、分单制作、舱单管理、报关操作。
8）后续规划（预研）
主单制作：抬头（收/发/通知）、代理信息、港口/航班、重量体积计费、运费及附加费、账单状态、接口发送（仓/EasyCargo 等）。结构为主表 + 子表。
分单制作：一主多分（或直单），与货主的合同维度。
舱单：汇总主/分单，形成托盘/位置映射并发送航司。
EDI：优先对接中间数据中心（如易飞瑞特），以一次对接覆盖多航司；本系统专注业务数据统一与财务一体化。
